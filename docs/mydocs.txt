💾 프로세스 핵심: 'AI의 외부 기억장치' 활용 전략
Gemini Web/CLI의 메모리(컨텍스트)가 날아가도 작업을 이어갈 수 있는 유일하고 가장 확실한 방법은, 모든 결정과 코드 상태를 'Git 리포지토리'와 'GEMINI.md' 파일에 기록하는 것입니다.

1. 📚 지식 저장소: GEMINI.md와 Git (메모리 대체)

역할,저장소,설명
장기 기억 (규칙/목표),GEMINI.md,"**최종 목표, ECS 아키텍처 규칙, 모든 CRITICAL RULE (특히 이벤트 시스템 규칙)**을 저장합니다. 이것이 Web/CLI Gemini에게 **'프로젝트의 영구적인 기억'**을 심어주는 핵심입니다."
단기 기억 (최근 변경 사항),Git & history.txt,"CLI에서 코드를 수정하면 Git으로 커밋하고, 최근 변경 사항 요약을 history.txt에 기록합니다."
코드 상태,Git 리포지토리,프로젝트의 실제 코드 상태는 항상 Git에 저장됩니다.

2. 🔁 메모리가 날아갔을 때, '작업 이어가기' 프로세스
Gemini Web 세션이 종료되어 메모리가 초기화되었을 때, 다음 단계를 순서대로 수행하여 작업을 정확히 이어갈 수 있습니다.
단계,수행할 작업,지휘자 프롬프트 예시 (Web UI)
  1. 컨텍스트 재주입,GitHub 링크와 GEMINI.md의 내용을 재주입하여 AI에게 프로젝트의 '기억'을 상기시킵니다.,"""이 GitHub 리포지토리 [링크]의 코드와, 내 GEMINI.md 파일에 정의된 모든 CRITICAL RULE을 기반으로 분석을 시작해 줘."""
  2.  코드 분석/확인,AI에게 현재의 코드 상태와 수행할 작업을 명확히 확인하도록 지시합니다.,"""현재 dungeon/system.py에서 MovementSystem의 상태를 확인하고, 이벤트 시스템 통합 작업이 어디까지 진행되었는지 분석해 줘."""
  3. 설계/디버그 요청,분석 결과를 바탕으로 다음 단계의 설계 또는 디버그 작업을 요청합니다.,"""분석 결과에 따라, GEMINI.md의 이벤트 시스템 규약을 위반하는 부분을 수정하는 CLI 작업 계획서를 만들어 줘."""
